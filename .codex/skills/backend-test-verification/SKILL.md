# 后端测试验证

## 使用目标

证明后端改动可运行、符合需求、没有明显权限/租户/数据一致性问题。

## 验证流程

1. 根据改动类型选择验证范围。
2. 优先运行：
   - `pnpm run build`
   - `pnpm run test`
   - `pnpm run lint`
3. 有接口改动时，补充接口级验证。
4. 有数据库改动时，验证 migration、实体映射和关键查询。
5. 有 `dbsql/` 改动时，检查 SQL 文件命名、用途说明、幂等性、回滚方案和执行顺序。
6. 有权限/租户改动时，验证跨租户访问被拒绝或查不到数据。
7. 有库存/订单改动时，验证事务、库存流水、异常回滚和边界数量。

## 测试建议

- Service 复杂业务逻辑写单元测试。
- Controller、Guard、认证流程、核心业务链路考虑 e2e。
- 导入类逻辑测试成功、失败、部分失败、重复数据、空文件和非法模板。
- 分页接口测试筛选、排序、页码和空结果。
- 权限测试普通用户、租户管理员、平台管理员和未登录访问。

## 验证清单

- 构建是否通过。
- 测试是否通过。
- lint 是否引入新问题。
- API 返回结构是否符合契约。
- DTO 校验是否生效。
- `dbsql/` 是否包含必要 schema/data/rollback SQL。
- 租户隔离是否有效。
- 事务失败是否回滚。
- 敏感字段是否未返回。
- Swagger 是否同步。

## 最终汇报

- 说明跑了哪些命令和结果。
- 说明没有跑的验证项及原因。
- 说明仍需人工验证的环境依赖，例如 MySQL、Redis、OSS、短信服务。
